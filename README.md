## 📖 项目愿景

**ModuAI** 旨在打造下一代“零代码、低认知”的智能硬件敏捷开发生态。

传统开源硬件（如 Arduino）存在电路设计复杂、代码门槛高的问题，而现有的模块化方案（如 M5Stack）又缺乏 AI 辅助的需求拆解能力。ModuAI 首创 **「AI 自然语言交互 ➡️ 标准化模组智能匹配 ➡️ Web 端无代码组装与仿真 ➡️ 底层控制代码生成」** 的三位一体全链路辅助模式。

我们致力于填补“专业级硬件平台”与“低龄启蒙教具”之间的空白，让技术小白、高校学生以及中小微企业，都能以极低的试错成本，将脑海中的物联网与硬件创意快速转化为可落地的工程原型。

---

## 🎯 行业痛点与 ModuAI 破局方案

### 痛点分析
1. **传统开源生态割裂（如 Arduino）**：拥有海量生态，但要求开发者精通 C/C++ 与底层电路（接线、焊接），对“技术小白”极不友好，试错成本高昂。
2. **模块化方案缺乏“大脑”（如 M5Stack）**：虽解决了物理拼插问题，但从“用户需求”到“硬件选型”依然依赖人工经验，缺乏智能化辅助。
3. **无代码套件扩展性弱**：市面部分图形化编程硬件偏向 K12 启蒙，无法向下兼容复杂的工业流，难以满足高阶科创与企业敏捷打样的需求。

### 破局之道：软硬整装与 AI 赋能
ModuAI 并不旨在推翻现有开源生态，而是定位为**现有开发流的“效能加速器”**。通过将庞杂的硬件选型工作交由 AI 引擎处理，结合概念标准化的磁吸/接口规范，极大缩短从 MVP（最小可行性产品）构思到实物落地的周期。

## 🚀 核心特性

* **🧠 AI 智能需求解析引擎**
  突破传统的查阅手册选型模式。用户只需输入自然语言（例：“帮我设计一个阳台植物缺水报警器”），AI 即可自动拆解意图，精准映射出 100% 协议兼容的硬件模组清单（BOM）。
* **🌌 高保真 Web 仿真工作台**
  软件先行的极致体现。基于 WebGL 与现代前端框架构建沉浸式、带有毛玻璃高级质感（Apple Pro Aesthetic）的工作台。支持 3D/2.5D 元器件自由拖拽、磁吸拼插演示，在购买实体硬件前即可完成物理拓扑验证。
* **💻 工业级底层代码实时生成**
  告别从零手写代码。基于前端工作台的拼插状态，右侧面板会实时演算并生成包含 `#include` 依赖与 `Setup/Loop` 逻辑的 C++/Arduino 标准代码，支持一键复制烧录。
* **🧱 概念标准化模组体系 (Hardware Roadmap)**
  制定统一的软硬件接口规范（如通用磁吸接口+功能标准化定义标签 `TH-001`）。平台生成的方案可无缝对接未来 ModuAI 实体套件或现有兼容生态，真正实现“电子积木式”的即插即用。

## 🤝 目标用户与适用场景

* **🎓 高校理工科学生（如电子、通信、物联网低年级）**：作为科创课程、大创项目的辅助工具，帮助快速建立“硬件创意→实现”的系统认知，打破“PPT 创新”的局限。
* **💡 创客爱好者（无专业硬件背景）**：作为低成本验证想法的“创意跳板”，无需深厚电路知识即可完成智能家居、DIY 设备的搭建。
* **🏢 中小微企业基层员工**：无需供养庞大的硬件研发团队，即可快速搭建轻量级场景监测原型（如门店客流统计、仓库温湿度预警）。

## 🗺️ 项目演进路线 (Roadmap)

ModuAI 采取“轻资产、重服务”的敏捷迭代战略：

* **🚩 阶段一：工具赋能（Web 端软硬整装模拟器）—— [当前阶段]**
  聚焦纯软件 SaaS 形态。依托高校大创孵化，完成核心 UI/UX 与 AI 拆解逻辑的验证。用户无需实体硬件即可在线完成验证闭环，低成本积累“场景-模组”匹配的数字飞轮。
* **🚩 阶段二：教育服务与社区生态**
  切入高校基础科创教学，提供“平台 SaaS + 概念模组教学套件（合作代工）”的软硬一体教具方案；建立开源创客社区，沉淀海量 UGC 硬件设计案例。
* **🚩 阶段三：商业下沉与开放生态**
  面向中小企业开放技术接口（API/SDK），允许第三方外设接入。提供从 AI 方案生成到 PCBA 敏捷打样、企业级物联网对接的一站式服务，成为开源硬件领域的底层分发基础设施。

## ⚙️ 架构与技术栈

### 软件架构 (Web MVP)
* **前端框架**: React 18, TypeScript, Vite
* **UI/视觉**: Tailwind CSS v4, Lucide Icons, 高级毛玻璃材质体系
* **3D 引擎与渲染**: Three.js, `@react-three/fiber`, `@react-three/drei`
* **状态管理与逻辑**: Zustand, 本地 JSON 结构化规则引擎 (LLM API Ready)

### 硬件生态规范 (规划中)
* **主控兼容**: 默认兼容 ESP32、Arduino 系列架构底座
* **通信总线**: 标准化 I2C, UART, GPIO 引脚定义抽象
* **物理形态**: 磁吸防呆设计 + 标准插拔排针接口

## 🔧 安装与本地部署

**1. 克隆项目到本地:**
```bash
git clone <你的远程仓库地址>
cd moduAI
```

**2. 安装工程依赖:**
```bash
npm install
```

**3. 置入 3D 成品模型文件:**
系统默认内置了高端的光影方块作为模型兜底。要想获得极致真实的物理元器件质感，请将下载好的 `.glb` 格式 3D 模型放入 `public/models/` 目录。名称约定如下：
* `public/models/esp32.glb`
* `public/models/dht11.glb`
* `public/models/oled.glb`

**4. 启动开发服务器:**
```bash
npm run dev
```
然后在浏览器中打开 `http://localhost:5173` 即可体验！
